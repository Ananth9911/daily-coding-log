        How to Determine Time Complexity

        1. The Hierarchy (Best to Worst)

        Memorize this order. If your solution is O(N²) and the array size is 10^5, it will Time Out (TLE). You must move left on this scale.

        O(1) Constant ➔ O(log N) Logarithmic ➔ O(N) Linear ➔ O(N log N) Log-Linear ➔ O(N²) Quadratic ➔ O(2^N) Exponential ➔ O(N!) Factorial

        2. The 3 Golden Rules (How to calculate it)

        • Rule 1: Drop the Constants. If your code loops through an array twice sequentially, it is `O(N + N) = O(2N)`. You drop the `2`. The answer is just `O(N)`.
        • Rule 2: Drop the Non-Dominant Terms. If your code has a nested loop `O(N²)` followed by a single loop `O(N)`, the total is `O(N² + N)`. As N gets massive, the single N stops mattering. Drop it. The answer is `O(N²)`.
        • Rule 3: Different Inputs = Different Variables. If you iterate through Array A (size N) and then Array B (size M), the time complexity is `O(N + M)`, NOT `O(N)`. Do not merge them unless the problem explicitly states N = M.

        3. The Visual Cues (How to spot it in code)

        Look at the structural shape of your loops to instantly guess the Big-O.

        for (int i = 0; i < N; i++) { ... }

        ➔ O(N): Standard iteration. Touches every element once.

        for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) { ... }
        }

        ➔ O(N²): A loop inside a loop, both going to N. (Classic Brute Force).

        for (int i = 0; i < N; i++) {
        for (int j = i + 1; j < N; j++) { ... }
        }

        ➔ O(N²): Even though `j` starts at `i`, mathematically this runs (N * (N-1)) / 2 times. Drop constants = O(N²).

        while (left < right) { ... left++; right--; }

        ➔ O(N): Two pointers moving towards each other. They cover the array exactly once collectively.

        for (int i = 1; i < N; i *= 2) { ... }

        ➔ O(log N): If the loop variable multiplies or divides (like Binary Search cutting the space in half), it's logarithmic.

        Arrays.sort(nums);

        ➔ O(N log N): The instant you call a standard sorting library, your time complexity can never be better than N log N.

        return dfs(node.left) + dfs(node.right);

        ➔ O(2^N): Basic recursion that branches into two paths every step (without memoization/caching).

        4. The "Amortized" Trap (SDE-2 Interview Secret)

        If you see a `while` loop inside a `for` loop, you will instantly want to say `O(N²)`. Stop and look at the variables.

        for (int right = 0; right < N; right++) {
        while (sum > target) { left++; }
        }

        ➔ O(N): This is a Sliding Window. The `right` pointer moves N times. The `left` pointer moves a maximum of N times across the entire execution of the program. Total operations = 2N. This is called Amortized O(N).
